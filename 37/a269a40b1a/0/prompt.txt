# Claude Code Task: Make strali.solutions crawlable for bots, LLMs, and fetchers

## Problem

strali.solutions is a React SPA (client-side rendered). When any bot, LLM, or HTTP fetcher requests a page, they only get the empty `index.html` shell with `<div id="root"></div>` — no actual content. This affects:

- Search engine crawlers (Google can render JS but others can't)
- LLM fetchers (Claude, ChatGPT, Perplexity) — they see nothing
- Social media preview crawlers (LinkedIn, Twitter, Facebook)
- Any `curl` or `fetch()` call to the site

The site has `react-snap` in package.json and a `build` script that runs it, but the Docker build uses `build:no-snap` which skips prerendering. There's also an `seo-issue.md` in the repo documenting these problems.

## Goal

Implement **static prerendering at build time** so that every route serves full HTML content to any fetcher, while still working as a React SPA for interactive users (hydration).

## Current Architecture

- **Framework:** React 18 + Vite + React Router (BrowserRouter)
- **Server:** Express (server.js) serving `dist/` with SPA fallback (`app.get('*', ...)`)
- **Content:** All text lives in `src/context/LanguageContext.jsx` as a translations object (DE + EN)
- **Language:** Auto-detected via `navigator.language` (client-side), toggle button for DE/EN
- **Deployment:** Docker → Synology NAS → Cloudflare Tunnel
- **Existing but unused:** `react-snap` is a dependency, `npm run build` calls it but Docker uses `build:no-snap`

## Routes to prerender

From `App.jsx`:
```
/              → Homepage (Hero + Services + AIPlatforms + About + SelectedPublicWork + Contact)
/impressum     → Legal notice / Impressum
/datenschutz   → Privacy policy
/assessment    → Free IT Assessment booking page
```

Plus a 404 fallback page.

## Requirements

### 1. Prerender all routes in BOTH languages

Each route must be prerenderable in German (default) AND English. The prerendered HTML must contain the **full text content** so that a simple `curl https://strali.solutions/` returns readable content including services, about text, AI platforms section, etc.

**Language strategy for prerendering:**
- Default prerendered HTML should be **German** (primary audience is Austrian businesses)
- English content should also be accessible. Options (pick the best one):
  - Option A: Prerender German only, English loads via JS hydration (simpler)
  - Option B: Prerender both `/de/...` and `/en/...` route prefixes (better for SEO but requires route changes)
  - Option C: Use `<noscript>` or hidden content blocks for both languages
- **Recommend and implement the option that gives the best crawlability with the least refactoring.**

### 2. Fix react-snap or replace it

`react-snap` is already in package.json but:
- It hasn't been maintained since 2020
- The Docker build skips it (`build:no-snap`)
- It may not work with React 18 + Vite

**Evaluate and decide:**
- Can `react-snap` work reliably with the current stack? If yes, fix the config and enable it in the Docker build.
- If not, use an alternative like `vite-plugin-prerender`, `prerender-spa-plugin`, or a custom puppeteer/playwright script.
- Whatever you choose, it must work in the Docker multi-stage build (Alpine + Node 22).

### 3. Update server.js to serve prerendered HTML

The Express server currently does a blind SPA fallback. After prerendering:
- Serve the prerendered HTML file for each known route
- The React app should **hydrate** on top of the prerendered HTML (not re-render from scratch)
- Ensure `react-helmet-async` meta tags from prerendering are in the served HTML

### 4. Update sitemap.xml and robots.txt

- Ensure `public/sitemap.xml` includes all prerendered routes with correct `<lastmod>` dates
